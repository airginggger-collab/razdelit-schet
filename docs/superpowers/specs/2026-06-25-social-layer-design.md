**Статус:** утверждён
**Базируется на:** [2026-06-24-razdelit-schet-design.md](2026-06-24-razdelit-schet-design.md) (MVP)

## Что строим

Надстройка над MVP, превращающая приложение из «калькулятора дележа» в инструмент сбора денег: организатор указывает телефоны участников, персонально напоминает каждому сумму через WhatsApp/Telegram, сохраняет частых друзей и видит историю прошлых счетов с возможностью вернуться к любому.

Четыре связанные фичи:
1. Телефоны участников + персональная отправка «что оплатить».
2. Deep-link каналы: WhatsApp (по номеру), Telegram (share), системный Share.
3. Частые друзья (сохранение/выбор имени+номера).
4. Экран истории с восстановлением счёта.

**Ограничение, унаследованное от MVP:** без сервера. Вся доставка — через deep-link (ты жмёшь «отправить» в открывшемся мессенджере), всё хранение — localStorage. Реал-тайм-синхронизации между устройствами нет.

## Фича 1 — Телефоны участников + персональная отправка

### Данные
В объект участника добавляется поле `phone` (строка, опционально):

```js
{ id, name, color, share, paid, paidAmount, phone }
```

### Экран 2 (Участники)
Строка участника получает второе поле — телефон — и иконку 📇 (выбор из друзей):

```
А  [Алия__________]  [+7 700 …]  📇
Б  [Болат_________]  [+7 701 …]  📇
```

- Телефон опционален. Аватар-инициал по-прежнему от имени.
- 📇 открывает пикер друзей (Фича 3).

### Экран 4 (Трекер)
Карточка участника получает кнопку **«Напомнить»**. Тап открывает мини-меню каналов (Фича 2). Кнопка есть всегда; набор каналов зависит от наличия номера.

### Текст напоминания

```
Привет, {имя}! С тебя за счёт: {сумма} ₸.
Отметить оплату: {ссылка}
```

- `{сумма}` — `participant.share`, форматирование `ru-RU`.
- `{ссылка}` — deep-link сессии (та же, что у кнопки «Поделиться»): открывший видит трекер.
- Без имени — «Привет!».

## Фича 2 — Deep-link каналы

Построители живут в `js/share.js`. Телефон нормализуется: убираются все не-цифры; ведущая `8` → `7` (конвенция KZ/RU); для `wa.me` номер без `+`.

| Канал | Условие | URL |
|---|---|---|
| WhatsApp | есть `phone` | `https://wa.me/<digits>?text=<enc(text)>` |
| Telegram | всегда | `https://t.me/share/url?url=<enc(url)>&text=<enc(text_без_url)>` |
| Поделиться | всегда (если есть `navigator.share`) | `navigator.share({ text, url })` |

- Без номера WhatsApp в меню скрыт.
- Telegram по номеру напрямую невозможен (ограничение Telegram) — только share-диалог, где ты выбираешь чат.
- Каждый канал открывается через `window.open(url, '_blank')` (или `location.href` на мобильном), системный — через `navigator.share`.

## Фича 3 — Частые друзья

### Данные
localStorage, ключ `razdelit-friends`:

```js
[ { name: 'Алия', phone: '77001234567' } ]
```

Ключ уникальности — нормализованный `phone`. Друг без номера не сохраняется (нечем напоминать).

### Пикер
📇 на строке участника открывает список сохранённых друзей чипами:

```
📇 Из друзей:
· Алия  · Болат  · Дина
```

- Тап по чипу → автозаполняет имя+телефон этой строки.
- Долгое нажатие на чип → подтверждение → удаление из друзей.
- Пусто → «Нет сохранённых».

### Автосохранение
При нажатии «Далее» на экране 2 каждый участник с непустыми именем И телефоном дописывается/обновляется в друзьях (upsert по номеру). Молча, без диалога.

## Фича 4 — История + восстановление

### Доступ
Кнопка **«📜 История»** в шапке экрана 1 (рядом с заголовком).

### Экран истории
Новый экран (вне линейного потока 1–4). Список записей из `razdelit-history`, новые сверху:

```
📜 История
25 июн · 6 000 ₸ · ✅ собрано
24 июн · 12 400 ₸ · ⏳ 8 200 / 12 400
```

- Статус: `✅ собрано` если собрано ≥ итога, иначе `⏳ X / Y`.
- Дата — `toLocaleDateString('ru-RU', { day:'numeric', month:'short' })`.
- Тап по записи → загружает сессию в `state` → открывает трекер (экран 4).
- «Назад» → экран 1.
- Пусто → «Пока счетов нет».

### Пересмотр хранения истории (важно)

Текущая реализация (после багфикса 2026-06-25) сохраняет историю один раз при 100% сбора. Для показа **незавершённых** счетов это меняется на **upsert по `id` сессии**:

- У сессии появляется `state.id`: присваивается при входе на трекер (`initScreen4`), если `id` ещё нет — `crypto.randomUUID()` с фолбэком на `Date.now()+rand`. Так покрыты все пути входа (ручной поток, открытие по ссылке, восстановление из истории сохраняют свой `id`).
- `saveHistory()` ищет запись с этим `id` и **обновляет** её; если нет — добавляет сверху. Хранятся последние 10.
- Вызывается на каждый `updateTotals()` — прогресс персистится, но **одна запись на счёт** (дубли невозможны).
- Конфетти по-прежнему один раз за счёт (флаг `completed`).

Это заменяет вчерашний подход «сохранять один раз в конце» — он чинил дубли, но терял незавершённые счета. Upsert чинит и то, и другое.

## Приватность

`encodeSession()` кодирует всю сессию в URL. С телефонами это утечка — номера попадут в расшаренную ссылку. Поэтому при кодировании **поле `phone` вырезается** из каждого участника. Телефоны живут только в localStorage на устройстве организатора и в исходящих deep-link (которые ты отправляешь явно), но не в расшаренном состоянии.

## Архитектура / файлы

| Файл | Изменение | Ответственность |
|---|---|---|
| `js/store.js` | **новый** | localStorage: друзья (list/upsert/remove) + история (upsert/list/get). Единый слой хранения. |
| `js/share.js` | расширить | deep-link построители (whatsapp/telegram/текст) + вырезание `phone` при `encodeSession`. |
| `js/app.js` | изменить | поле телефона + пикер на экране 2; кнопка «Напомнить» + меню на экране 4; init экрана истории; `session.id`; saveHistory → upsert. |
| `index.html` | изменить | кнопка «📜 История» на экране 1; шаблон `tmpl-screen-history`. |
| `style.css` | дополнить | чипы друзей, мини-меню каналов, поле телефона, список истории. |
| `sw.js` | bump → `razdelit-v4` | добавить `./js/store.js` в ASSETS. |

### Роутер
`showScreen(n)` сейчас ждёт число и шаблон `tmpl-screen${n}`. Расширяется до приёма строкового id (`'history'` → `tmpl-screen-history`). Для именованных экранов init берётся из карты по ключу, «Назад» истории жёстко ведёт на экран 1 (а не `n−1`).

### Изоляция
- `store.js` ничего не знает про DOM — чистые функции над localStorage. Тестируется независимо.
- `share.js` — чистые построители строк (URL/текст), без побочных эффектов. Тестируется независимо.
- `app.js` — оркестрация и DOM.

## Граничные случаи

- **Восстановление завершённого счёта** → трекер сразу показывает всех оплативших + «Новый счёт».
- **Участник без номера** → «Напомнить» показывает только Telegram/Поделиться.
- **Пустая история / пустые друзья** → плейсхолдеры.
- **Невалидный номер** (буквы/мусор) → нормализация оставит цифры; если пусто — WhatsApp скрыт.
- **`navigator.share` отсутствует** (десктоп) → канал «Поделиться» скрыт, остаются WhatsApp/Telegram.
- **Старые записи истории без `id`** (с прошлой версии) → при загрузке трактуются как самостоятельные; новый upsert их не ломает (просто не находит по id → пишет новую при следующем сборе).

## Что НЕ входит

- Реал-тайм-синхронизация между устройствами (нужен сервер).
- Автоматическая отправка без участия пользователя (нужен сервер/бот/SMS-API).
- Telegram-доставка напрямую по номеру (ограничение платформы).
- Редактирование друзей (только добавить/удалить).
- Группы друзей, аватары друзей, импорт из контактов телефона.
